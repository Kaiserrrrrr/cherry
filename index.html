Introduction

Cherry.js is a zero-config, "healing" engine designed to fix common Lighthouse audit failures in real-time. It patches accessibility gaps, SEO omissions, and performance bottlenecks without requiring changes to your legacy codebase.

Installation

Include the minified script at the end of your <head> for the best results:

<script src="dist/cherry.js"></script>

What it fixes

  • Accessibility: Automatically generates aria-labels, fixes tabindex abuse, and repairs contrast ratios.
  • Performance: Forces lazy-loading on off-screen assets and prevents Cumulative Layout Shift (CLS) by injecting missing dimensions.
  • SEO: Dynamically generates missing H1 tags, Meta descriptions, and lang attributes.
  • Best Practices: Secures target="_blank" links and repairs invalid DOM nesting.

Usage

Once loaded, Cherry.js initializes an IdleCallback. It waits for the browser to be "bored" before performing DOM surgery, ensuring that it never contributes to Total Blocking Time (TBT).
